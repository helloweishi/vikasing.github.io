NiceText

A Java based webpage text extraction lib

What is NiceText?

Its a small Java based library to extract main content from a web page, it removes HTML, JS, CSS and other text clutter from an HTML document. It extracts the main text from a webpage based on text density of the different part of the webpage and later clustering these regions to find the largest and most appropriate region.

How to use NiceText?

Right now you can download 2 files NiceTextType.java and HTMLHelper.java, include them in your project and do the following:

HTMLHelper htmlHelper = new HTMLHelper();       
NiceTextType niceTextType = htmlHelper.getNiceText("http://www.nytimes.com/2013/06/11/us/how-edward-j-snowden-orchestrated-a-blockbuster-story.html?pagewanted=all&_r=0");
System.out.println(niceTextType.getArticleText());
System.out.println(niceTextType.getNiceText());
System.out.println(niceTextType.getLargestTextBlock());

I'm still working on the improvements and tweaks, with the addition of new Classes above code is most likely to change.

How does it work?

NiceText finds the text density of different regions on a webpage and tries to determine the largest region buy using clustering. This large region or cluster is chosen as the maintext or niceText.

I'll write a detailed post on how it works sometime in future.

There is also a method getArticleText() which just tries to find out an article region by looking at the tag names, class names and ids, e.g. it looks for following words: article|section|content|page|post|... It sometimes produces bad results but is faster than the getNiceText().

You may also try getAllText() to get all the text of a webpage.
